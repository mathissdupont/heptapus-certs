# HeptaCert Geliştirici Dokümantasyonu

HeptaCert; etkinlik yönetimi, kayıt, check-in, **doğrulanabilir dijital sertifika** üretimi ve katılımcı CRM'ini tek platformda birleştirir. Bu dokümantasyon, platformu kendi sistemlerinize bağlamanız için gereken her şeyi içerir: REST API, MCP (AI agent) entegrasyonu, CLI aracı ve webhook sistemi.

## Dört Entegrasyon Yolu

import { Cards, Card } from "nextra/components";

HeptaCert'e dört farklı yoldan erişebilirsiniz. Hepsi aynı API anahtarını ve aynı [scope modelini](/api-reference/authentication#scopelar-i̇zinler) paylaşır — hangi yolu seçerseniz seçin yetkiler aynı kalır.

| Yol | En uygun olduğu durum |
|-----|----------------------|
| **REST API** | Sunucu-sunucu entegrasyon, web/mobil arka uç, scriptler |
| **MCP Agent** | Claude, Cursor gibi AI asistanlarına HeptaCert yetkisi vermek |
| **CLI (`hc`)** | Terminal otomasyonu, CI/CD, hızlı yönetim |
| **Webhooks** | Olaylar gerçekleştiğinde sisteminizi anında haberdar etmek |

## Neler Yapabilirsiniz?

- **Etkinlik yönetimi** — etkinlik oluştur, güncelle, kaydı aç/kapat
- **Katılımcı yönetimi** — tek tek veya toplu (CSV/Excel) ekle, ara, dışa aktar
- **Sertifika üretimi** — tekil veya yüzlerce sertifikayı toplu, asenkron üret; iptal et; kademe yönet
- **Check-in & devam** — oturum bazlı manuel/QR check-in, devam analitiği
- **Otomasyon** — "sertifika yayımlanınca e-posta gönder" gibi olay-tabanlı kurallar
- **CRM** — katılımcıları etkinlikler arası izle, lead skorla, mükerrerleri birleştir
- **Webhooks** — sertifika/CRM olaylarını dış sistemlere imzalı POST ile ilet

## Base URL

```
https://heptacert.com/api
```

Tüm istekler HTTPS üzerinden yapılmalıdır. HTTP istekleri otomatik olarak reddedilir. Makine-okunur OpenAPI spesifikasyonu `https://heptacert.com/api/openapi.json` adresindedir.

## Kimlik Doğrulama (Özet)

Her istekte API anahtarınızı Bearer token olarak gönderin:

```bash
curl https://heptacert.com/api/admin/events \
  -H "Authorization: Bearer hc_live_YOUR_KEY"
```

Ayrıntılar için [Kimlik Doğrulama](/api-reference/authentication) sayfasına bakın.

## Versiyonlama ve Stabilite

Mevcut API stabildir ve versiyonlanmamıştır. Geriye dönük uyumluluğu korumaya öncelik veririz; kırıcı bir değişiklik gerekirse e-posta ile önceden bildirim yapılır. Yeni alanlar ve endpoint'ler herhangi bir zamanda **eklenebilir** — istemcileriniz tanımadığı alanları yok sayacak şekilde toleranslı (tolerant reader) yazılmalıdır.
